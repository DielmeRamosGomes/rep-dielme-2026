O QUE SÃO REQUISITOS FUNCIONAIS

Requisitos funcionais descrevem o que o sistema deve fazer. 
Eles definem as funcionalidades, comportamentos e serviços que o 
sistema oferece aos usuários, atendendo às suas necessidades.

O QUE SÃO REQUISITOS NÃO FUNCIONAIS

Requisitos não funcionais descrevem como o sistema deve funcionar.
Eles estão relacionados à qualidade, desempenho, segurança, 
usabilidade e outras restrições do sistema.

EXEMPLOS DE REQUISITOS PARA UM SISTEMA E-COMMERCE

Requisitos Funcionais:
1. O sistema deve permitir que o usuário se cadastre.
2. O sistema deve permitir que o cliente realize compras e 
pagamentos online.

Requisitos Não Funcionais:
1. O sistema deve carregar as páginas em até 3 segundos.
2. O sistema deve garantir a segurança dos dados dos usuários.

DIFERENÇA ENTRE ELICITAÇÃO E DOCUMENTAÇÃO DE REQUISITOS

Elicitação de requisitos é o processo de levantar e compreender as
necessidades dos stakeholders por meio de entrevistas, reuniões e
questionários. Documentação de requisitos é o processo de registrar essas 
necessidades de forma clara e organizada.

OS QUATRO VALORES DO MANIFESTO ÁGIL

1. Indivíduos e interações mais que processos e ferramentas.
2. Software em funcionamento mais que documentação abrangente.
3. Colaboração com o cliente mais que negociação de contratos.
4. Responder a mudanças mais que seguir um plano.

PRINCÍPIOS DO MANIFESTO ÁGIL RELACIONADOS À ENTREGA DE VALOR

Princípio 1: “Nossa maior prioridade é satisfazer o cliente
através da entrega contínua e adiantada de software com valor
agregado.”

Princípio 3: “Entregar frequentemente software funcionando,
de poucas semanas a poucos meses, com preferência à menor
escala de tempo.”

Princípio 7: “Software funcionando é a medida primária
do progresso.”

DIFERENÇA ENTRE ERRO, DEFEITO E FALHA

Erro é um engano humano cometido durante o desenvolvimento.
Defeito é um problema inserido no sistema como consequência de um
erro.
Falha é quando o sistema apresenta um comportamento incorreto 
durante a execução.

O QUE SÃO TESTES UNITÁRIOS

Testes unitários são testes realizados para verificar o 
funcionamento correto de pequenas partes do sistema, como funções 
ou métodos, de forma isolada.
